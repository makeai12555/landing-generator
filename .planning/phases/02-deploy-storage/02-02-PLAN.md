**Step 1: Create Blob Store**

2. Click "Create Store" -> "Blob"
3. Name it (e.g., "courseflow-blob") and connect it to the `courseflow-landing` project
4. The `BLOB_READ_WRITE_TOKEN` env var is auto-added to the project

**Step 2: Set Environment Variables**
1. Go to https://vercel.com/makeai12555s-projects/courseflow-landing/settings/environment-variables
2. Add these variables for the **Production** environment (copy values from your `.env.local`):
   - `SESSION_SECRET`
   - `AUTH_SCRIPT_URL`
   - `GEMINI_API_KEY`
   - `APPS_SCRIPT_URL`
   - `GEMINI_IMAGE_MODEL` (set to `gemini-3-pro-image-preview`)
3. Do NOT set `NEXT_PUBLIC_BASE_URL` — the app now uses `VERCEL_PROJECT_PRODUCTION_URL` which Vercel provides automatically

**Step 3: Verify Vercel plan tier**
- Check if the team is on Hobby or Pro plan
- If Hobby: banner generation will timeout (10s limit). Pro is needed for the 60s maxDuration.
- Note: the app will still work for everything except banner generation on Hobby tier

**Step 4: Push to master to trigger deploy**
Once env vars are set, type "ready" and Claude will push to master to trigger the auto-deploy.
