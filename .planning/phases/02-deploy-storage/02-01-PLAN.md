<objective>
Migrate all file-system storage to Vercel Blob and fix production URL resolution.

Purpose: The app currently uses `fs.writeFile`/`fs.readFile` which silently fail on Vercel's read-only serverless filesystem. This plan replaces all fs calls with `@vercel/blob`, extracts base64 images into separate public blobs (keeping JSON small), fixes the `NEXT_PUBLIC_BASE_URL` hardcoded fallback, adds `maxDuration` to the banner route, and captures `instructorEmail` from the session for future Phase 5 use.

Output: All storage operations use Vercel Blob; base URL detection works on Vercel; banner route won't timeout; landing JSON includes instructor email.
</objective>

2. Create `landing-next/lib/storage.ts` with three exported functions:

   **`saveLanding(id: string, data: object): Promise<void>`**
   - Uses `put(`landings/${id}.json`, JSON.stringify(data), { access: 'private', contentType: 'application/json', allowOverwrite: true })`
   - Import `put` from `@vercel/blob`

   **`getLanding(id: string): Promise<LandingPageData | null>`**
   - Uses `head(`landings/${id}.json`)` to get the blob metadata
   - Then `fetch(meta.url)` to get the JSON
   - Catches `BlobNotFoundError` and returns `null`
   - Import `head` from `@vercel/blob`

   **`saveImage(path: string, base64DataUri: string): Promise<string>`**
   - Strips the `data:image/...;base64,` prefix
   - Converts to Buffer via `Buffer.from(base64, 'base64')`
   - Uses `put(path, buffer, { access: 'public', contentType: 'image/png', allowOverwrite: true })`
   - Returns the public blob URL string

   Also add a **`getBaseUrl(): string`** helper:
   - If `process.env.VERCEL_PROJECT_PRODUCTION_URL` is truthy, return `https://${process.env.VERCEL_PROJECT_PRODUCTION_URL}`
   - Otherwise return `process.env.NEXT_PUBLIC_BASE_URL || 'http://localhost:3000'`
   - Export this function

3. Update `landing-next/types/landing.ts`:
   - Add `instructorEmail?: string;` field to `LandingPageData` (after `sheetId`)
