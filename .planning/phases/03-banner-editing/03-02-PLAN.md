**Part A: Update BannerPreview.tsx**

Redesign BannerPreview to support independent refinement for each image. The component currently renders two cards (banner + background). Each card needs its own refinement flow.

**New props interface:**
```ts
interface BannerPreviewProps {
  bannerUrl?: string;
  backgroundUrl?: string;
  isLoading?: boolean;      // Keep for initial generation (Phase A)
  status?: string;          // Keep for initial generation status
  // NEW — refinement callbacks
  onBannerReplace?: (newDataUri: string, colors: { primary: string; accent: string }) => void;
  onBackgroundReplace?: (newDataUri: string) => void;
  bannerRefinementsLeft?: number;   // e.g. 4, 3, 2, 1, 0
  backgroundRefinementsLeft?: number;
  originalBannerUrl?: string;       // Phase A original for revert
  originalBackgroundUrl?: string;   // Phase A original for revert
  onBannerRevert?: () => void;
  onBackgroundRevert?: () => void;
}
```

**Per-image refinement block (render below each image card):**

For each image (banner and background), add a refinement section that appears when the image exists and refinementsLeft > 0:

1. **Input field** — a single-line `<input>` with:
   - `dir="rtl"` for Hebrew
   - Rotating placeholder from a list of Hebrew examples. Use `useState` + `useEffect` with a 3-second `setInterval` that cycles through examples:
     - Banner examples: `["תכהה קצת את הרקע", "הגדל את הכותרת", "תוסיף מעט צבע כחול", "תשנה את הסגנון ליותר מודרני", "הוסף יותר ניגודיות"]`
     - Background examples: `["תכהה את הרקע", "תוסיף גוונים חמים", "תעשה את התמונה יותר בהירה", "הוסף אפקט טשטוש עדין", "שנה את הצבעים לכחול"]`
   - Submit button (send icon) — disabled when input is empty or loading

2. **Loading overlay** — when refining THIS image:
   - An overlay div (absolute positioned over the image `aspect-[16/9]` container)
   - Semi-transparent backdrop (`bg-black/40`)
   - Spinner + "משפר תמונה..." text
   - Only this image shows loading — the other image remains interactive

3. **Preview mode** — after Gemini returns a refined image:
   - Show the NEW image in place of the current one (temporary preview state)
   - Two buttons below the image:
     - **"החלף תמונה"** (Replace image) — green/primary styled. Calls `onBannerReplace(newDataUri, colors)` or `onBackgroundReplace(newDataUri)`.
     - **"בטל"** (Cancel) — gray/outline styled. Discards the preview, restores previous image.
   - Counter decrements on BOTH Replace and Cancel (generation already happened, API cost incurred).

4. **Counter display** — small text below the input: "נותרו {N} שיפורים" (e.g. "נותרו 3 שיפורים"). When 0, hide the input and show "מיצית את מכסת השיפורים" (grayed out text).

5. **Revert button** — appears when current image differs from original:
   - Small link-style button: "חזור לתמונה המקורית"
   - Calls `onBannerRevert()` or `onBackgroundRevert()`
   - Does NOT reset the refinement counter (per CONTEXT.md)

**Internal state per image** (use `useState` within BannerPreview):
- `bannerRefineInput: string` — current input value
- `backgroundRefineInput: string` — current input value
- `bannerRefining: boolean` — loading state for banner refinement
- `backgroundRefining: boolean` — loading state for background refinement
- `bannerPreview: string | null` — temporary preview data URI (null = no preview)
- `backgroundPreview: string | null` — temporary preview data URI
- `bannerPreviewColors: { primary: string; accent: string } | null` — colors from preview

**Fetch logic** (inside BannerPreview):
When instructor submits an instruction for an image:
```ts
const response = await fetch("/api/refine-banner", {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({
    imageDataUri: currentImageUrl,  // the current banner/background URL
    instruction: inputValue,
    isBanner: true/false,           // true for banner, false for background
  }),
});
const result = await response.json();
if (result.ok) {
  // Set preview state — don't replace yet
  setPreview(result.image);
  if (result.colors) setPreviewColors(result.colors);
}
```

**Keep existing functionality intact:**
- The download button on the banner card — keep it, always downloads the current (non-preview) banner
- The `isLoading` / `status` props for Phase A initial generation — keep working
- The background fallback to bannerUrl — keep

**Part B: Update /create/config/page.tsx**

Add state management for refinement:

1. **New state variables:**
```ts
const MAX_REFINEMENTS = 4;
const ORIGINALS_KEY = "courseDataOriginals";

const [bannerRefinementsLeft, setBannerRefinementsLeft] = useState(MAX_REFINEMENTS);
const [backgroundRefinementsLeft, setBackgroundRefinementsLeft] = useState(MAX_REFINEMENTS);
```

2. **Store originals on mount** — in the existing `useEffect` that loads from localStorage:
   - After parsing courseData, check if `localStorage.getItem(ORIGINALS_KEY)` exists.
   - If NOT: save the current `banner_url` and `background_url` as originals:
     ```ts
     localStorage.setItem(ORIGINALS_KEY, JSON.stringify({
       banner_url: data.generated_assets.banner_url,
       background_url: data.generated_assets.background_url,
     }));
     ```
   - If exists: load the originals for revert functionality.
   - Store in state: `const [originals, setOriginals] = useState<{ banner_url?: string; background_url?: string } | null>(null);`

3. **Callback: handleBannerReplace(newDataUri, colors)**
   - Update `courseData.generated_assets.banner_url = newDataUri`
   - Update `courseData.branding.theme.colors.primary = colors.primary` and `accent = colors.accent`
   - Save updated courseData to localStorage
   - Decrement `bannerRefinementsLeft`
   - Update state with `setCourseData`

4. **Callback: handleBackgroundReplace(newDataUri)**
   - Update `courseData.generated_assets.background_url = newDataUri`
   - Save updated courseData to localStorage
   - Decrement `backgroundRefinementsLeft`
   - Update state with `setCourseData`

5. **Callback: handleBannerRevert()**
   - Restore `courseData.generated_assets.banner_url` from originals
   - Save to localStorage
   - Update state — do NOT reset counter

6. **Callback: handleBackgroundRevert()**
   - Restore `courseData.generated_assets.background_url` from originals
   - Save to localStorage
   - Update state — do NOT reset counter

7. **Callback: handleRefinementUsed(type: 'banner' | 'background')**
   - Called on Cancel (preview discarded but generation still counted)
   - Decrements the appropriate counter

8. **Pass new props to BannerPreview:**
```tsx
<BannerPreview
  bannerUrl={assets?.banner_url}
  backgroundUrl={assets?.background_url}
  onBannerReplace={handleBannerReplace}
  onBackgroundReplace={handleBackgroundReplace}
  bannerRefinementsLeft={bannerRefinementsLeft}
  backgroundRefinementsLeft={backgroundRefinementsLeft}
  originalBannerUrl={originals?.banner_url}
  originalBackgroundUrl={originals?.background_url}
  onBannerRevert={handleBannerRevert}
  onBackgroundRevert={handleBackgroundRevert}
/>
```

9. **Clean up originals on landing creation** — in `createLandingPage`, after successful creation, remove the originals key:
   ```ts
   localStorage.removeItem(ORIGINALS_KEY);
   ```
