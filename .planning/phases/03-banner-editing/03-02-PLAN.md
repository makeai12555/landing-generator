**Step A: Create `landing-next/components/course/RefinementPanel.tsx`**

A `"use client"` component with the following behavior:

**Props interface:**
```typescript
interface RefinementPanelProps {
  currentBannerUrl?: string;       // from courseData.generated_assets.banner_url
  currentBackgroundUrl?: string;   // from courseData.generated_assets.background_url
  courseTitle: string;              // from courseData.course_details.title
  courseSubtitle?: string;          // from courseData.course_details.description (first 80 chars)
  isRefining: boolean;             // loading state lifted to parent
  onSubmit: (instruction: string) => void;  // triggers API call in parent
  previewBanner?: string | null;   // base64 preview after generation
  previewBackground?: string | null;
  onAccept: () => void;            // parent handles state + localStorage update
  onReject: () => void;            // parent reverts preview
  onRevert: () => void;            // parent restores original Step 1 banner
  refinementsUsed: number;         // how many used so far
  refinementLimit: number;         // cap (5)
  hasOriginal: boolean;            // whether original banner differs from current (show revert button)
}
```

**Internal state:**
- `instruction: string` — the input field value
- `placeholderIndex: number` — cycles through PLACEHOLDER_EXAMPLES on a 3-second interval while input is empty

**PLACEHOLDER_EXAMPLES array:**
```typescript
const PLACEHOLDER_EXAMPLES = [
  "תעשה את הרקע כהה יותר",
  "שנה את הצבעים לטונים חמים",
  "הוסף אפקט מקצועי יותר",
  "תעשה את הסגנון מינימליסטי",
  "שנה לתאורת לילה",
];
```

**Placeholder rotation:**
- Use `useEffect` with a 3-second `setInterval` to cycle `placeholderIndex` when `instruction` is empty.
- Clear interval when `instruction` has content or component unmounts.

**UI layout** (rendered below the banner preview area):

1. **Refinement input section** (inside a card matching the existing page style: `bg-white rounded-2xl p-6 shadow-sm border border-gray-200`):
   - Header: "שיפור באנר" (bold, same style as other section headers)
   - Subtext: remaining count — `נותרו {remaining} שיפורים` (if remaining > 0) or `הגעת למגבלת השיפורים` (if 0)
   - Text input: `<input type="text">` with rotating placeholder, RTL, full width. Disabled when `isRefining`, `remaining === 0`, or when preview is showing (previewBanner is set).
   - Submit button: "שפר באנר" with a magic wand icon (`auto_fix_high` material symbol). Disabled when `instruction` is empty, `isRefining`, `remaining === 0`, or preview is active. On click: call `onSubmit(instruction)` then clear the instruction field.

2. **Preview action buttons** (only shown when `previewBanner` is set and `!isRefining`):
   - Two buttons side by side:
     - Accept (green-ish): "אשר" with `check` icon → calls `onAccept()`
     - Reject (gray/red-ish): "דחה" with `close` icon → calls `onReject()`
   - Style: prominent, centered, with clear visual distinction

3. **Revert button** (only shown when `hasOriginal` is true and no preview is active):
   - "חזור לבאנר המקורי" — secondary button style, below the input area
   - Calls `onRevert()`

**Styling:** Use Tailwind classes consistent with the existing page design. RTL direction. Hebrew text.

**Step D: Integrate into `landing-next/app/create/config/page.tsx`**

Add the following state to `LandingConfigPage`:
```typescript
const [isRefining, setIsRefining] = useState(false);
const [previewBanner, setPreviewBanner] = useState<string | null>(null);
const [previewBackground, setPreviewBackground] = useState<string | null>(null);
const [previewColors, setPreviewColors] = useState<{ primary: string; accent: string } | null>(null);
const [refinementsUsed, setRefinementsUsed] = useState(0);
const [originalBannerUrl, setOriginalBannerUrl] = useState<string | null>(null);
const [originalBackgroundUrl, setOriginalBackgroundUrl] = useState<string | null>(null);
```

**REFINEMENT_LIMIT** constant: `5` (at Claude's discretion within 3-5 range per CONTEXT.md).

**On mount** (in the existing useEffect after parsing courseData):
- Save the initial banner as original: `setOriginalBannerUrl(data.generated_assets.banner_url || null)` and same for background. These are the "Step 1" originals that "Revert" restores to.

**handleRefineSubmit(instruction: string):**
1. `setIsRefining(true)`
2. Call `fetch("/api/refine-banner", { method: "POST", headers: { "Content-Type": "application/json" }, body: JSON.stringify({ instruction, currentBannerUrl: assets.banner_url, currentBackgroundUrl: assets.background_url, courseTitle: details.title, courseSubtitle: details.description.slice(0, 80) }) })`
3. Parse response. If `result.ok`:
   - `setPreviewBanner(result.banner)`, `setPreviewBackground(result.background)`, `setPreviewColors(result.colors)`
   - Increment: `setRefinementsUsed(prev => prev + 1)`
4. If error: alert with Hebrew error message, do NOT increment count.
5. `setIsRefining(false)`

**handleAccept():**
1. Update courseData with preview images and colors:
   ```typescript
   const updated = {
     ...courseData,
     generated_assets: {
       ...courseData.generated_assets,
       banner_url: previewBanner,
       background_url: previewBackground,
     },
     branding: {
       ...courseData.branding,
       theme: {
         ...courseData.branding.theme,
         colors: previewColors ? { primary: previewColors.primary, accent: previewColors.accent } : courseData.branding.theme.colors,
       },
     },
   };
   ```
2. `setCourseData(updated)`, `localStorage.setItem(STORAGE_KEY, JSON.stringify(updated))`
3. Clear preview: `setPreviewBanner(null)`, `setPreviewBackground(null)`, `setPreviewColors(null)`

**handleReject():**
1. Clear preview state: `setPreviewBanner(null)`, `setPreviewBackground(null)`, `setPreviewColors(null)`
2. (Count was already incremented on submit — rejections count toward limit per CONTEXT.md)

**handleRevert():**
1. If `originalBannerUrl` exists:
   ```typescript
   const updated = {
     ...courseData,
     generated_assets: { ...courseData.generated_assets, banner_url: originalBannerUrl, background_url: originalBackgroundUrl || courseData.generated_assets.background_url },
   };
   ```
2. `setCourseData(updated)`, `localStorage.setItem(STORAGE_KEY, JSON.stringify(updated))`
3. Clear preview state
4. Do NOT reset `refinementsUsed` — reverting still counts per CONTEXT.md

**In the JSX:**

Pass `overlayLoading={isRefining}` and `overlayStatus="משפר באנר..."` to `BannerPreview`.

When `previewBanner` is set and `!isRefining`, show the preview banner in BannerPreview instead of the current one:
```tsx
<BannerPreview
  bannerUrl={previewBanner || assets?.banner_url}
  backgroundUrl={previewBackground || assets?.background_url}
  overlayLoading={isRefining}
  overlayStatus="משפר באנר..."
/>
```

Render `<RefinementPanel>` below the `<BannerPreview>` in the right column, only when `assets?.banner_url` exists (no refinement if no banner generated yet):
```tsx
{assets?.banner_url && (
  <RefinementPanel
    currentBannerUrl={assets.banner_url}
    currentBackgroundUrl={assets.background_url}
    courseTitle={details.title}
    courseSubtitle={details.description?.slice(0, 80)}
    isRefining={isRefining}
    onSubmit={handleRefineSubmit}
    previewBanner={previewBanner}
    previewBackground={previewBackground}
    onAccept={handleAccept}
    onReject={handleReject}
    onRevert={handleRevert}
    refinementsUsed={refinementsUsed}
    refinementLimit={REFINEMENT_LIMIT}
    hasOriginal={originalBannerUrl !== null && originalBannerUrl !== assets.banner_url}
  />
)}
```
