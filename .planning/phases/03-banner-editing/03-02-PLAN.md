**1. Create `components/landing/RefinementPanel.tsx`** — a `"use client"` component.

Props interface:
```typescript
interface RefinementPanelProps {
  landingId: string;
  currentBannerUrl?: string;
  currentBackgroundUrl?: string;
  originalBannerUrl?: string;
  originalBackgroundUrl?: string;
  refinementsUsed: number;
  refinementLimit: number;
  courseTitle: string;
  courseSubtitle: string;
  heroRef?: React.RefObject<HTMLElement | null>;  // ref to the Hero section for loading overlay positioning
}
```

State:
- `isOpen` (boolean) — panel open/closed
- `instruction` (string) — input value
- `isLoading` (boolean) — Gemini generating
- `preview` (null | { banner: string, background: string, colors: { primary: string, accent: string } }) — preview data from API
- `used` (number) — initialized from `refinementsUsed` prop
- `currentBanner` (string) — tracks the live banner URL (starts from prop, updates on accept)
- `currentBackground` (string) — same for background
- `hasOriginal` (boolean) — derived: `!!originalBannerUrl` or `used > 0 && first accept happened`
- `placeholderIndex` (number) — for rotating placeholder
- `showHeroOverlay` (boolean) — true while Gemini is generating, used to render overlay on the hero area

Computed: `remaining = refinementLimit - used`, `canRefine = remaining > 0`.

**Rotating placeholder:** Array of 5 Hebrew examples:
```typescript
const PLACEHOLDERS = [
  "תעשה את הרקע כהה יותר",
  "שנה את הצבעים לטונים חמים",
  "הוסף אפקט מקצועי לתמונה",
  "תעשה את הסגנון מינימליסטי יותר",
  "שנה לתאורת לילה",
];
```
Use `useEffect` with a 3-second `setInterval` to cycle `placeholderIndex` while `instruction` is empty. Clear interval when input has text or component unmounts.

**UI structure (RTL — use `dir="rtl"` on the panel):**

A. **Closed state:** Two fixed-position buttons in the bottom-left corner (RTL convention — Hebrew users expect actions on the left):
   - **Edit bubble:** Paintbrush or edit icon (SVG inline). Round button with primary color background, shadow, hover scale animation. Show a small badge with remaining count.
   - **Download button:** Positioned directly above the edit bubble (same left offset, ~60px above). Round button with secondary/neutral background, download arrow icon (SVG inline). On click, triggers `handleDownload` (see handlers below). Always downloads the latest `currentBanner` URL. Only rendered when `currentBannerUrl` is truthy.

B. **Open state (panel):** Fixed-position panel, bottom-left, ~350px wide, white background, rounded corners, shadow-xl. Contains:
   - Header row: "עריכת באנר" title + remaining count badge ("נותרו X שינויים") + close X button
   - If `canRefine`: textarea input (2 rows, auto-resize) with rotating placeholder, RTL direction, plus a Submit button (primary color, disabled when empty or loading)
   - If `!canRefine`: disabled message "ניצלת את כל השינויים האפשריים"
   - If `hasOriginal` or `originalBannerUrl`: "חזור לבאנר המקורי" revert button (outline style, not primary)
   - Loading state: when `isLoading`, show a spinner next to "יוצר באנר חדש..." text, disable input

D. **Hero overlay (loading):** When `showHeroOverlay` is true, render a `createPortal`-based overlay (or absolute-positioned `<div>` if `heroRef` is provided) that covers the hero/banner section of the landing page. The overlay should be semi-transparent white/dark (`bg-black/40`), centered spinner (animated SVG or Tailwind `animate-spin`), and "...יוצר באנר חדש" text below the spinner. Implementation: if `heroRef?.current` exists, use `createPortal(<div className="absolute inset-0 z-40 flex flex-col items-center justify-center bg-black/40 rounded-xl">...</div>, heroRef.current)`. The hero section's container must have `position: relative` for this to work — this will be ensured in the landing page update (see step 3 below). Set `showHeroOverlay = true` alongside `isLoading = true` in handleSubmit, and set both to false when generation completes or errors.

C. **Preview state (when `preview` is not null):** Instead of the input area, show:
   - The refined banner as an `<img>` tag with `src={preview.banner}` (this is a data:URI)
   - Two buttons side by side: "אשר ✓" (green/primary, calls accept handler) and "דחה ✗" (red/outline, calls reject handler)

**Handlers:**

`handleSubmit`:
- Set `isLoading = true`, `showHeroOverlay = true`, clear `preview`
- `fetch("/api/refine-banner", { method: "POST", headers: { "Content-Type": "application/json" }, body: JSON.stringify({ landingId, instruction, currentBannerUrl: currentBanner, currentBackgroundUrl: currentBackground, courseTitle, courseSubtitle }) })`
- On success: set `preview` from response JSON, increment `used` by 1, clear `instruction`
- On error: show alert with error message (keep it simple — no toast library needed)
- Set `isLoading = false`, `showHeroOverlay = false`

`handleAccept`:
- `fetch(`/api/landing/${landingId}`, { method: "PATCH", headers: { "Content-Type": "application/json" }, body: JSON.stringify({ action: "accept", bannerUrl: preview.banner, backgroundUrl: preview.background, theme: preview.colors, refinementsCount: used }) })`
- On success: update `currentBanner` and `currentBackground` state, clear `preview`, reload the page with `window.location.reload()` to show updated banner in Hero (simplest approach — the Hero is a server component that reads from Blob)
- On error: alert

`handleReject`:
- Just clear `preview`. Counter already incremented on submit. No API call needed.

`handleDownload`:
- Fetch `currentBanner` URL as a blob: `const res = await fetch(currentBanner); const blob = await res.blob();`
- Create a temporary download link: `const url = URL.createObjectURL(blob); const a = document.createElement("a"); a.href = url; a.download = "banner.png"; document.body.appendChild(a); a.click(); document.body.removeChild(a); URL.revokeObjectURL(url);`
- This downloads the latest accepted banner version as a PNG file — the instructor can then share it via WhatsApp.

`handleRevert`:
- `fetch(`/api/landing/${landingId}`, { method: "PATCH", headers: { "Content-Type": "application/json" }, body: JSON.stringify({ action: "revert" }) })`
- On success: `window.location.reload()`
- On error: alert

**Styling:** Use Tailwind classes throughout. The panel should have `z-50` to float above page content. Use smooth transitions for open/close (`transition-all duration-300`).

```typescript
const cookieStore = await cookies();
const token = cookieStore.get(SESSION_COOKIE_NAME)?.value;

```

Since the landing page is a server component but the RefinementPanel needs a ref to the Hero section for the loading overlay, create a thin client wrapper component inline (or as part of the same file). The approach:

- Wrap the Hero section's containing `<div>` with `position: relative` (add `relative` to its Tailwind classes) and attach a `ref` via a client component wrapper.
- The simplest approach: make the landing page body a client component that receives serialized data from the server component. Alternatively, wrap just the hero + refinement panel in a small client component `LandingClientShell`:

```tsx
// Inside page.tsx, add a "use client" component (can be in the same file above the server component,
// or extracted to a separate file — keep it inline for simplicity if short):
"use client";
function LandingClientShell({ data, isInstructor }: { data: LandingPageData; isInstructor: boolean }) {
  const heroRef = useRef<HTMLDivElement>(null);
  return (
    <>
      <div ref={heroRef} className="relative">
        <Hero {...heroProps} />
      </div>
      {isInstructor && (
        <RefinementPanel
          landingId={data.id}
          currentBannerUrl={data.assets.bannerUrl}
          currentBackgroundUrl={data.assets.backgroundUrl}
          originalBannerUrl={data.refinements?.originalBannerUrl}
          originalBackgroundUrl={data.refinements?.originalBackgroundUrl}
          refinementsUsed={data.refinements?.count ?? 0}
          refinementLimit={data.refinements?.limit ?? 5}
          courseTitle={data.course.title}
          courseSubtitle={data.course.description}
          heroRef={heroRef}
        />
      )}
    </>
  );
}
```

The exact implementation may vary — the key requirement is that `heroRef` points to the Hero section's wrapper so `createPortal` can render the overlay there. If making a separate client component is cleaner, extract it to `components/landing/LandingClientShell.tsx` and add to `files_modified`.

Note: `courseSubtitle` uses `data.course.description` since the `LandingPageData` type uses `description` rather than `subtitle_he`. This is the Hebrew subtitle content displayed on the banner.
