refinements?: {

**2. Create `app/api/refine-banner/route.ts`** — a new POST endpoint modeled closely on the existing `/api/banner/route.ts`.

Route config (top of file):
```typescript
export const runtime = "nodejs";
export const dynamic = "force-dynamic";
export const maxDuration = 60;  // Gemini takes 15-30s per image
```

Request body shape:
```typescript
{ landingId: string, instruction: string, currentBannerUrl: string, currentBackgroundUrl: string, courseTitle: string, courseSubtitle: string }
```

Implementation:
- Validate required fields. Return 400 if missing.
- Instantiate `GoogleGenAI` with `process.env.GEMINI_API_KEY`. Return 500 if missing.
- Use `const MODEL = process.env.GEMINI_IMAGE_MODEL || "gemini-3-pro-image-preview"`.
- **Fetch the current banner image** from `currentBannerUrl` server-side, convert to base64 via `Buffer.from(arrayBuffer).toString("base64")`.
- **Fetch the current background image** from `currentBackgroundUrl`, same conversion.
- Build two Gemini calls in parallel with `Promise.all`:
  - **Banner refinement:** Parts array with `{ text: bannerPrompt }` and `{ inlineData: { mimeType: "image/png", data: bannerBase64 } }`. The `bannerPrompt` should be: `"${instruction}\n\nCRITICAL: Keep all Hebrew text exactly as it appears. The title is: \"${courseTitle}\". The subtitle is: \"${courseSubtitle}\". Only change the visual elements described in the instruction above. Do NOT remove, change, or garble any Hebrew text."`. Config: `{ responseModalities: ["TEXT", "IMAGE"] }`.
  - **Background refinement:** Parts array with `{ text: bgPrompt }` and `{ inlineData: { mimeType: "image/png", data: bgBase64 } }`. The `bgPrompt` should be: `"${instruction}\n\nThis is the background version with NO TEXT. Apply the same visual change described above. Keep it a clean background suitable for text overlay. ABSOLUTELY NO TEXT in the output."`. Config: `{ responseModalities: ["TEXT", "IMAGE"] }`.
- Extract images from both responses using `extractImageFromResponse()` — copy this helper function from `/api/banner/route.ts` (it is not exported). If either returns null, return 502 with error message.
- Extract colors from the refined banner using the same `node-vibrant` pattern as `/api/banner/route.ts`: import `Vibrant` from `"node-vibrant/node"`, create buffer from bytes, get palette, extract primary (Vibrant.hex) and accent (DarkVibrant.hex).
- Return JSON: `{ ok: true, banner: "data:image/png;base64,...", background: "data:image/png;base64,...", colors: { primary, accent } }`.
- Wrap in try/catch, return 502 on error (same pattern as existing banner route).

**Do NOT** add auth checking to this route — the UI component (Plan 02) only renders for instructors, and the PATCH route (Task 2) handles auth for persistence. This keeps the refinement route simple (generate-only, no side effects).

  <action>
Add a `PATCH` export to the existing `/api/landing/[id]/route.ts` file. Keep the existing `GET` export unchanged.

Add required imports at the top:
```typescript
import { cookies } from "next/headers";
import { verifySessionToken, SESSION_COOKIE_NAME } from "@/lib/auth";
import { saveLanding } from "@/lib/storage";  // already imported: getLanding
```

**PATCH handler implementation:**

```typescript
export async function PATCH(
  req: Request,
  { params }: { params: Promise<{ id: string }> }
) {
```

1. **Auth check:** Read session cookie via `await cookies()`, get `SESSION_COOKIE_NAME` value, verify with `verifySessionToken()`. Return 401 if no token or invalid session.

2. **Load landing:** `const landing = await getLanding(id)`. Return 404 if not found.

3. **Authorization:** Compare `landing.instructorEmail` with `session.username`. Return 403 if they don't match. This ensures only the creating instructor can edit.

4. **Parse body:** Expect JSON with shape:
```typescript
{ action: "accept" | "revert", bannerUrl?: string, backgroundUrl?: string, theme?: { primary: string, accent: string }, refinementsCount?: number }
```

5. **Handle "accept" action:**
   - Read existing refinements: `const existing = landing.refinements ?? { count: 0, limit: 5 };`
   - On first-ever accept (when `!existing.originalBannerUrl`), capture originals: `originalBannerUrl = landing.assets.bannerUrl`, `originalBackgroundUrl = landing.assets.backgroundUrl`.
   - Otherwise preserve existing originals.
   - Build updated landing: spread `...landing`, override `assets: { bannerUrl: body.bannerUrl, backgroundUrl: body.backgroundUrl }`, override `theme: body.theme ?? landing.theme`, set `refinements: { count: body.refinementsCount, limit: existing.limit, originalBannerUrl, originalBackgroundUrl }`.
   - Call `await saveLanding(id, updated)`.
   - Return `Response.json({ ok: true })`.

6. **Handle "revert" action:**
   - Check `landing.refinements?.originalBannerUrl` exists. Return 400 if no original to revert to.
   - Build updated landing: spread `...landing`, override `assets: { bannerUrl: landing.refinements.originalBannerUrl, backgroundUrl: landing.refinements.originalBackgroundUrl }`.
   - Do NOT reset `refinements.count` (reverting still costs — per user decision).
   - Call `await saveLanding(id, updated)`.
   - Return `Response.json({ ok: true })`.

7. **Unknown action:** Return 400 with error message.

Wrap the entire handler in try/catch, return 500 on unexpected errors.
