Create `landing-next/app/api/refine-banner/route.ts` with the following:

**Route config** (same pattern as `/api/banner/route.ts`):
```ts
export const runtime = "nodejs";
export const dynamic = "force-dynamic";
export const maxDuration = 60;
```

**Request body interface:**
```ts
interface RefineRequest {
  imageDataUri: string;   // "data:image/png;base64,..." OR a Vercel Blob URL
  instruction: string;    // Hebrew free-text instruction (e.g. "תכהה את הרקע")
  isBanner: boolean;      // true = banner with text, false = background (no text)
}
```

**Implementation steps:**

1. Validate `GEMINI_API_KEY` env var (same as banner route).
2. Parse JSON body, validate `imageDataUri`, `instruction` (non-empty string), and `isBanner` (boolean) are present. Return 400 if missing.
3. Resolve the image to base64 bytes:
   - If `imageDataUri` starts with `"data:"` — strip the prefix, extract mimeType and base64 data directly.
   - If it's a URL (starts with `"http"`) — fetch it, convert to base64, detect mimeType from Content-Type header.
4. Build the Gemini request using `@google/genai` (`GoogleGenAI`):
   - Use the same `MODEL` constant pattern: `process.env.GEMINI_IMAGE_MODEL || "gemini-3-pro-image-preview"`
   - Contents array with TWO parts in a single user message:
     - Part 1: `{ inlineData: { mimeType, data: base64String } }` — the current image
     - Part 2: `{ text: promptText }` — the editing instruction
   - Config: `{ responseModalities: ["TEXT", "IMAGE"] }`
5. Build `promptText` based on `isBanner`:
   - If `isBanner === true`: "Edit the following Hebrew course banner image according to this instruction: \"{instruction}\". CRITICAL: Preserve all existing Hebrew text exactly as it appears — do not change, remove, or alter any text. Only modify the visual aspects as instructed. Return the modified image."
   - If `isBanner === false`: "Edit the following background image according to this instruction: \"{instruction}\". This is a background image with NO text. Do not add any text, letters, or characters. Only modify the visual aspects as instructed. Return the modified image."
6. Extract the image from response using the same `extractImageFromResponse` helper (copy it from banner/route.ts — it's small, no need for a shared module).
7. If `isBanner === true`, run color extraction using `node-vibrant` (same `extractColorsFromImage` pattern from banner/route.ts — copy the function). Return colors in response.
8. Return JSON response:
   ```ts
   { ok: true, image: "data:image/png;base64,...", colors?: { primary, accent } }
   ```
   - `colors` only present when `isBanner === true`.
9. On error, return `{ ok: false, error: message }` with status 502.

**Key patterns to reuse from `/api/banner/route.ts`:**
- `extractImageFromResponse` function (copy verbatim)
- `extractColorsFromImage` function (copy verbatim — uses `Vibrant.from(buffer).getPalette()`)
- Same imports: `GoogleGenAI` from `@google/genai`, `Vibrant` from `node-vibrant/node`
- Same error handling pattern

**Do NOT:**
- Import from `/api/banner/route.ts` (keep routes self-contained)
- Accept two images or return two images — this is strictly one-in, one-out
- Add conversation history or multi-turn — single request/response
