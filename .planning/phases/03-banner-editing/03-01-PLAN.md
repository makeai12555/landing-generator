Create `landing-next/app/api/refine-banner/route.ts` modeled closely on the existing `/api/banner/route.ts`.

**Route config (top of file):**
```
export const runtime = "nodejs";
export const dynamic = "force-dynamic";
export const maxDuration = 60;
```

**Request body schema:**
```typescript
interface RefineBannerRequest {
  instruction: string;          // Hebrew free-text instruction (e.g. "תעשה את הרקע כהה יותר")
  currentBannerUrl: string;     // URL or data:image/png;base64,... of current banner
  currentBackgroundUrl: string; // URL or data:image/png;base64,... of current background
  courseTitle: string;           // Hebrew course title — used to tell Gemini to preserve it
  courseSubtitle?: string;       // Hebrew subtitle — same reason
}
```

**Implementation steps:**

1. Validate request: `instruction`, `currentBannerUrl`, `currentBackgroundUrl`, and `courseTitle` are required. Return 400 if missing.

2. Check `GEMINI_API_KEY` env var. Return 500 if missing.

3. **Fetch current images and convert to base64:**
   - For each URL: if it starts with `data:image`, extract the base64 portion after the comma. If it's a remote URL (Vercel Blob), `fetch()` the URL, get `arrayBuffer()`, convert with `Buffer.from(buf).toString("base64")`.
   - Helper: `async function imageUrlToBase64(url: string): Promise<string>` — handles both data URIs and remote URLs.

4. **Build refinement prompts:**
   - Banner prompt (with text):
     ```
     Refine this course banner image based on this instruction: "${instruction}"

     CRITICAL: Keep all Hebrew text EXACTLY as it appears in the image. The course title is "${courseTitle}"${courseSubtitle ? ` and subtitle is "${courseSubtitle}"` : ''}. Do NOT change, remove, or modify any text. Only change the visual elements described in the instruction.

     Maintain the same 16:9 aspect ratio. Keep it professional and suitable for course marketing.
     ```
   - Background prompt (without text):
     ```
     Refine this background image based on this instruction: "${instruction}"

     This is a BACKGROUND version (NO TEXT). Apply the same visual change as instructed. Keep it clean — no text, letters, or characters. Suitable for text overlay.

     Maintain the same 16:9 aspect ratio.
     ```

5. **Call Gemini in parallel** using `Promise.all`:
   - Banner call: `client.models.generateContent` with `contents: [{ role: "user", parts: [{ text: bannerPrompt }, { inlineData: { mimeType: "image/png", data: bannerBase64 } }] }]` and `config: { responseModalities: ["TEXT", "IMAGE"] }`.
   - Background call: same pattern with background prompt and background base64.
   - Use `const MODEL = process.env.GEMINI_IMAGE_MODEL || "gemini-3-pro-image-preview"` — same as banner route.

6. **Extract images from response** — copy the `extractImageFromResponse` function from `/api/banner/route.ts` (same logic: iterate candidates[0].content.parts, find inlineData, decode base64 to Uint8Array).

7. **Extract colors** — copy the `extractColorsFromImage` function from `/api/banner/route.ts` (uses `Vibrant.from(Buffer.from(imageBytes)).getPalette()`). Run on the refined banner bytes.

8. **Return response:**
   ```json
   {
     "ok": true,
     "banner": "data:image/png;base64,...",
     "background": "data:image/png;base64,...",
     "colors": { "primary": "#hex", "accent": "#hex" }
   }
   ```
   Same shape as `/api/banner` response — the frontend can treat both identically.

9. **Error handling:** Wrap in try/catch. If Gemini returns no image (extractImageFromResponse returns null), return 502 with descriptive error. Log errors to console.

**Important notes:**
- Do NOT import from `/api/banner/route.ts` — copy the two helper functions (`extractImageFromResponse`, `extractColorsFromImage`). These are small, self-contained, and importing from another route is fragile.
- Import `GoogleGenAI` from `@google/genai` and `Vibrant` from `node-vibrant/node` — both already installed.
- The route does NOT need auth checking — it's called from the client-side `/create/config` page which is already behind auth via proxy.ts.
